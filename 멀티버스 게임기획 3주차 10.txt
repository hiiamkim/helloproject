버전 관리 시스템 (VCS)
 - 문서나 설계도, 소스 코드 등의 변경점 관리하는 소프트웨어
 - 소프트웨어 작성할 때 변경점 관리하는 것은 매우 중요
 - 언리얼 엔진은 결국 하나의 소프트웨어 제작 플랫폼이기 때문에 담겨진
    모든 콘텐츠를 구분 없이 소프트웨어로 이해함

 - 변경점 관리 : 어떤 내용을 누가 작성, 어느 시점에 들어갔는지 확인
 - 버전 관리 : 특정 시점에 꼬리표를 달아 표시 브랜치 등으로 동시에 여러 버전 개발 가능
 - 백업 복구 : 무언가 잘못되었을 때 특정시점으로 돌아가게, 사고로 날아가도 복구 가능
 - 협업 : 협업 가능

바이너리 파일이란
 - 실행할 수 있는 대부분의 파일이 바이너리 파일
 - 바이너리화 된 파일
 - txt는 바이너리화 안됨

로컬버전 관리
 - 서버가 별도로 있고 클라이언트가 중앙 서버에서 파일 받아 사용하는 방식
 - 로컬 VCS보다 관리가 쉽지만 중앙서버에 문제 발생하면 치명적
 - 대표적으로 RCS가있다.

분산버전관리 시스템
 - VCS 단순히 마지막 스냅샷을 체크아웃 하지 않고 저장소를 히스토리, 더불어 전부 복제
 - 서버에 문제가 생기면 이 복제물으로 다시 작업 시작 가능. 클라이언트 중 아무거나 골라도 서버 복원
 - 수많은 리모트 저장소 가질수 있기 때문에 다양한 방법으로 협업
